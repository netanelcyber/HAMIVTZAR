# Pie Semi Circle Chart

The Pie Semi Circle Chart displays proportional data in a semi-circular format, providing an elegant alternative to full pie charts with better space utilization for dashboards and compact layouts.

<Canvas of={ PieSemiCircleChartStories.Default } />

## Overview

The PieSemiCircleChart component renders data as segments in a semi-circular arc (180 degrees), making it ideal for displaying percentages, categories, or proportional breakdowns in a visually compact format. This chart type is particularly effective in dashboards where vertical space is limited.

<Source
	language="jsx"
	code={ `import { PieSemiCircleChart } from '@automattic/charts';
import '@automattic/charts/style.css';

	const data = [
		{
			label: 'MacOS',
			value: 30000,
			valueDisplay: '30K',
			percentage: 5,
		},
		{
			label: 'Linux',
			value: 22000,
			valueDisplay: '22K',
			percentage: 1,
		},
		{
			label: 'Windows',
			value: 80000,
			valueDisplay: '80K',
			percentage: 2,
		},
	];

	<PieSemiCircleChart
		data={ data }
		thickness={ 0.4 }
		label="OS"
		note="Windows +10%"
	/>` }
/>

The chart automatically validates data to ensure positive values and meaningful percentages, displaying error states for invalid data configurations.

## Error Handling

### Data Validation

The chart includes comprehensive data validation with user-friendly error messages:

<Canvas of={ PieSemiCircleChartStories.ErrorStates } />

**Validation Rules:**
- Data array cannot be empty
- Values and percentages must be non-negative
- Total percentage must be greater than 0 (unlike full pie charts, doesn't need to equal 100)

**Error States Handled:**
- **Empty Data**: Displays "No data available"
- **Negative Values**: Shows "Invalid data: Negative values are not allowed"
- **Zero Total**: Displays "Invalid percentage total: Must be greater than 0"

## Theming Integration

Pie Semi Circle Charts integrate seamlessly with the chart theming system. The default theme has neutral colors and styling, and is automatically applied to all charts unless a custom theme is provided. A custom theme can be provided by wrapping your chart in `GlobalChartsProvider` and passing a custom theme object with the properties you want to override to the `theme` prop:

<Source
	language="tsx"
	code={ `import { GlobalChartsProvider, PieSemiCircleChart, type ChartTheme } from '@automattic/charts';

	const customTheme: ChartTheme = {
		colors: ['#FF6B6B', '#4ECDC4', '#45B7D1', '#FFA07A'],
	};

	<GlobalChartsProvider theme={customTheme}>
		<PieSemiCircleChart data={data} />
	</GlobalChartsProvider>` }
/>

## Responsive Behavior

By default, charts **fill their parent container's dimensions** while maintaining a 2:1 width-to-height aspect ratio. The parent must have an explicit height:

<Canvas of={ PieSemiCircleChartStories.Default } />

<Source
	language="jsx"
	code={ `// Fill parent container (default) - parent needs explicit height
	<div style={{ width: '100%', height: '400px' }}>
		<PieSemiCircleChart data={ data } />
	</div>

	// Fixed dimensions - chart constrains to 2:1 ratio within these bounds
	<PieSemiCircleChart data={ data } width={ 600 } height={ 300 } />` }
/>

The chart always maintains a 2:1 width-to-height ratio. When both `width` and `height` are provided, the chart constrains to whichever dimension is more restrictive. Use `width` and/or `height` to constrain the chart to specific pixel dimensions.

<Canvas of={ PieSemiCircleChartStories.FixedDimensions } />

For more details on responsive behavior, see the [Responsive Design section](./?path=/docs/js-packages-charts-library-introduction--docs#responsive-design) in the introduction.

## Animation

The Pie Semi Circle Chart component supports an optional entry animation that creates a smooth reveal effect when the chart first renders:

<Canvas of={ PieSemiCircleChartStories.Animation } />

<Source
	language="jsx"
	code={ `<PieSemiCircleChart
		data={ data }
		animation={ true }
		label="OS"
		note="Windows +10%"
	/>` }
/>

### Animation Behavior

- **Opt-in**: Animation is disabled by default and must be explicitly enabled with the `animation` prop
- **Accessibility**: Automatically respects the user's `prefers-reduced-motion` system setting - animation will not play for users who prefer reduced motion
- **Effect**: Creates a radial wipe reveal effect that sweeps clockwise from the top, progressively revealing the chart segments
- **Duration**: 1000ms (1 second)

**Note**: The animation plays once when the chart initially renders and does not repeat.

## Migration

### From Full Pie Charts

Semi-circle charts use the same data format as full pie charts, making migration straightforward:

<Source
	language="jsx"
	code={ `// Full pie chart
	<PieChart
		data={ data }
		size={ 400 }
	/>

	// Semi-circle chart (uses width instead of size)
	<PieSemiCircleChart
		data={ data }
		width={ 400 }
	/>` }
/>

### Key Differences from Full Pie Charts

- **Aspect Ratio**: Semi-circle charts have a 2:1 width-to-height ratio
- **Space Efficiency**: Takes up approximately 50% less vertical space
- **Data Validation**: Percentages don't need to sum to 100 (allows for partial data representation)
- **Positioning**: Different internal positioning calculations for labels and legends

### Data Validation Differences

Unlike full pie charts that require percentages to sum to exactly 100:

<Source
	language="jsx"
	code={ `// Full pie chart - must sum to 100
	const fullPieData = [
		{ label: 'A', value: 60, percentage: 60 },
		{ label: 'B', value: 40, percentage: 40 }, // Total: 100%
	];

	// Semi-circle chart - flexible totals
	const semiCircleData = [
		{ label: 'Completed', value: 75, percentage: 75 },
		{ label: 'Remaining', value: 25, percentage: 25 }, // Total: 100%
	];

	// Or partial completion
	const partialData = [
		{ label: 'Progress', value: 60, percentage: 60 }, // Shows 60% of semi-circle
	];` }
/>
