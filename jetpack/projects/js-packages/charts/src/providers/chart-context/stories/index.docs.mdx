# Global Charts Context

The Global Charts Context provides centralized theme management, color coordination, and chart registration for multiple charts within an application. It ensures consistent styling and behavior across all chart components while optimizing performance through intelligent color caching and stable group assignments.

<Canvas of={ ChartContextStories.Default } />

## Overview

The Global Charts Context system consists of three main parts:

1. **GlobalChartsProvider** - A React provider component that wraps your application
2. **GlobalChartsContext** - The underlying React context that manages shared state
3. **Context hooks** - Utilities for accessing theme and registration functionality

## API Reference

For detailed information about provider props, context values, and type definitions, see the [API Reference](?path=/docs/js-packages-charts-library-global-context-api-reference--docs).

## Basic Usage

### Setting Up the Provider

Wrap your application or chart components with `GlobalChartsProvider` to enable theme sharing and color coordination:

<Source
	language="tsx"
	code={ `import { GlobalChartsProvider, LineChart, BarChart } from '@automattic/charts';

	function App() {
		return (
			<GlobalChartsProvider>
				<div className="dashboard">
					<LineChart data={ salesData } />
					<BarChart data={ conversionData } />
					{ /* All charts share the same theme and color coordination */ }
				</div>
			</GlobalChartsProvider>
		);
	}` }
/>

### Basic Provider Props

- **`theme`** *(optional)*: Partial theme object to customize chart appearance
- **`children`**: Chart components and other React elements

## Color Coordination

### Stable Colors for Groups

One of the key features of the Global Charts Context is **stable color assignment for groups**. When data series have a `group` property, the same group will always receive the same color across all charts, even when data changes:

**Try toggling the countries on/off in the Default story** - notice how each country maintains its assigned color regardless of which other countries are visible. This creates a consistent visual experience across multiple charts and data updates.

<Source
	language="tsx"
	code={ `// Data with group assignments

			label: 'Great Britain Sales',
			group: 'great-britain', // Different group, different stable color
			data: [ /* time series data */ ],

	];` }

## Chart Registration

### Automatic Registration

Charts automatically register themselves with the Global Charts Context when mounted. This enables features like:

- **Coordinated legends** - Multiple charts can share legend items
- **Color synchronization** - Ensures consistent colors across charts
- **Metadata sharing** - Charts can share contextual information

<Source
	language="tsx"
	code={ `// Charts automatically register with a unique ID

		data={ data }
		chartId="sales-trends" // Optional: provide explicit ID
	/>

	// The context tracks:
	// - Legend items from this chart
	// - Chart type ('line-chart')
	// - Any additional metadata` }

## Context Hooks

### useGlobalChartsContext

Access the full context for advanced chart functionality:

<Source
	language="tsx"
	code={ `import { useGlobalChartsContext } from '@automattic/charts';

	function CustomChart() {
		const {
			theme,
			getElementStyles,
			registerChart,
			unregisterChart,
			getChartData
		} = useGlobalChartsContext();

		// Get consistent styling for chart elements
		const elementStyles = getElementStyles( {
			data: myDataPoint,
			index: 0,
			overrideColor: undefined,
			legendShape: 'line'
		} );

		return (
			<div style={ { color: elementStyles.color } }>
				{ /* Custom chart implementation */ }
			</div>
		);
	}` }
/>

### useGlobalChartsTheme

Access just the theme when you only need styling information:

<Source
	language="tsx"
	code={ `import { useGlobalChartsTheme } from '@automattic/charts';

	function ThemeAwareComponent() {
		const theme = useGlobalChartsTheme();

		return (
			<div style={ {
				backgroundColor: theme.backgroundColor,
				color: theme.legend?.labelStyles?.color
			} }>
				Styled using global chart theme
			</div>
		);
	}` }
/>

### useChartRegistration

Register custom charts with the global context:

<Source
	language="tsx"
	code={ `import { useChartRegistration } from '@automattic/charts';

	function CustomChart( { data, chartId } ) {
		// Register this chart with the global context
		useChartRegistration( {
			chartId: chartId || 'custom-chart',
			legendItems: data.map( item => ( {
				label: item.label,
				color: item.color
			} ) ),
			chartType: 'custom-chart',
			isDataValid: data.length > 0,
			metadata: { customProperty: 'value' }
		} );

		return <div>{ /* Chart implementation */ }</div>;
	}` }
/>

## Performance Considerations

### Color Caching

The Global Charts Context implements intelligent color caching to optimize performance:

- **HSL conversion caching** - Color format conversions are cached and reused
- **Hue analysis** - Color relationships are pre-computed for better color selection
- **Group mapping persistence** - Group-to-color assignments persist across renders

### Efficient Updates

- **Stable references** - Context functions use `useCallback` for stable references
- **Selective re-renders** - Only charts that need updates are re-rendered when context changes
- **Deep comparison** - Legend items use deep comparison to prevent unnecessary updates

## Advanced Usage

### Custom Element Styling

Use the context's `getElementStyles` function to get consistent styling for custom chart elements:

<Source
	language="tsx"
	code={ `function CustomChartElement( { seriesData, index, isHighlighted } ) {
		const { getElementStyles } = useGlobalChartsContext();

		const styles = getElementStyles( {
			data: seriesData,
			index,
			overrideColor: isHighlighted ? '#FF0000' : undefined,
			legendShape: 'circle'
		} );

		return (
			<circle
				fill={ styles.color }
				stroke={ styles.lineStyles.stroke }
				strokeWidth={ styles.lineStyles.strokeWidth }
				r="5"
			/>
		);
	}` }
/>

## Accessibility

### Theme Color Considerations

When creating custom themes, ensure sufficient contrast between background and foreground colors to meet WCAG AA standards (4.5:1 for normal text, 3:1 for large text)

### Automatic Color Generation

When your data has more series than colors defined in your theme, the Global Charts Context automatically generates additional colors using a sophisticated algorithm:

#### Golden Ratio Distribution

The system uses the **golden ratio (φ ≈ 0.618)** to generate mathematically pleasing color distributions:

- **Hue spacing** - New hues are distributed for optimal visual separation
- **Hue range constraint** - Generated colors stay within an expanded range (130%) of your theme's existing colors
- **Wrap-around handling** - Intelligently handles cases where theme colors span across the 0° hue boundary

**Note that only hex colors are supported for generating colors.**

#### Accessibility-First Generation

All generated colors maintain WCAG AA compliance:

- **Base lightness: 35%** - Ensures 4.5:1 contrast ratio against white backgrounds
- **Lightness variations** - Creates 4 levels (35%, 43%, 51%, 59%) for variety while maintaining accessibility
- **Saturation variations** - Uses 3 levels (60%, 75%, 90%) for visual distinction

#### Perceptual Distance Optimization

The algorithm ensures visual distinction between colors:

- **Minimum distance threshold** - Maintains 25+ units of perceptual distance between colors
- **Smart retry logic** - Up to 50 attempts per color to find sufficiently different options
- **Distance calculation** - Uses weighted HSL distance optimized for chart readability

<Source

## Troubleshooting

### Common Issues

**Charts not inheriting theme:**
- Ensure charts are wrapped with `GlobalChartsProvider`
- Check that the theme object is properly structured

**Colors not staying consistent:**
- Verify that data points include consistent `group` properties
- Check for conflicting color overrides in individual chart data

**Performance issues:**
- Avoid creating new theme objects on every render
- Use `useMemo` for complex theme calculations
- Minimize frequent context value changes

### Error Boundaries

The context includes error handling for invalid usage, for example, accessing the context outside of a `GlobalChartsProvider`:

<Source
	language="tsx"
	code={ `// This will throw a helpful error

	}` }
