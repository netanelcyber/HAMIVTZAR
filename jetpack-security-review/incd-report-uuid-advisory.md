# דיווח לגורם רלוונטי — פגיעות תלות (Dependency Advisory) בכלי ה-CI של פרויקט Jetpack (Automattic)

**תאריך:** 2026-07-10
**מדווח:** בדיקת קוד סטטית + סריקת תלויות על גבי עותק מקומי (snapshot) של קוד המקור הפתוח של Jetpack, ענף `trunk`, מתוך המאגר הציבורי `github.com/Automattic/jetpack`.

## תקציר מנהלים

בסריקת תלויות JavaScript (`pnpm audit`) שבוצעה על גבי קוד המקור המלא של
תוסף הוורדפרס Jetpack (Automattic, פרויקט קוד פתוח, ~5 מיליון+ התקנות
פעילות), אותרה **פגיעות ידועה ומתועדת ציבורית ברמת חומרה בינונית
(Moderate)** בחבילת JavaScript בשם `uuid`, המשמשת אך ורק ככלי עזר
ל-CI (בדיקות/כיסוי קוד) ואינה חלק מהקוד המופץ בפועל למשתמשי הקצה
של התוסף.

זהו **ממצא היגיינת-תלויות שגרתי, לא ניצול פעיל ולא פרצה ב-0-day**.
מזהה ה-Advisory כבר פורסם ציבורית במאגר ה-GitHub Security Advisory
Database, וכולל תיקון זמין. הדיווח מוגש למטרות שקיפות/מודעות בלבד.

## פרטי הממצא

| שדה | ערך |
|---|---|
| חבילה פגיעה | `uuid@8.3.2` |
| מזהה Advisory | [GHSA-w5hq-g745-h8pq](https://github.com/advisories/GHSA-w5hq-g745-h8pq) |
| חומרה | Moderate |
| תיאור | חוסר בדיקת גבולות buffer (`buf` parameter) בפונקציות `v3`/`v5`/`v6` של החבילה |
| גרסאות פגיעות | `< 11.1.1` |
| גרסה מתוקנת | `>= 11.1.1` |
| שרשרת תלות | `.github/files/coverage-munger` → `nyc` → `istanbul-lib-processinfo` → `uuid` |
| היכן זה רץ בפועל | כלי כיסוי-קוד (`nyc`) בצנרת ה-CI/CD של הפרויקט בלבד |
| האם מגיע לקוד המופץ למשתמשי קצה | **לא** — תלות פיתוח (dev dependency) בלבד |

## הערכת סיכון

- **חשיפה בפועל: נמוכה מאוד.** התלות הפגיעה נמצאת אך ורק בכלי בדיקת
  כיסוי קוד שרץ בסביבת ה-CI הפנימית של המפתחים, ואינה נכללת בחבילת
  הפלאגין המותקנת באתרי וורדפרס.
- **אין אינדיקציה לניצול פעיל** (no known exploited-in-the-wild status)
  נכון למועד הבדיקה.
- מדובר בממצא שגרתי מסוג "יש לעדכן תלות טרנזיטיבית" — לא בפרצת אבטחה
  הדורשת תגובה דחופה.

## המלצה

עדכון שרשרת התלות (`nyc` / `istanbul-lib-processinfo`) לגרסה שמביאה
`uuid >= 11.1.1`, או הוספת override מפורש בקובץ ה-lock. זהו תיקון
היגיינה סטנדרטי, לא תיקון אבטחה דחוף.

## הערת מתודולוגיה

בנוסף לסריקת התלויות, בוצעה גם בדיקת קוד סטטית (semgrep, כללי אבטחה
ל-PHP/JS/TS) על מלוא קוד המקור של הפרויקט, וכן audit תלויות PHP
(`composer audit`) על פני 20 תת-פרויקטים. לא אותרה בבדיקות אלו כל
פגיעות ניתנת-לניצול נוספת. דוח מלא: `jetpack-security-review/REPORT.md`
במאגר זה.
